## Overview

This specification defines the requirements for renaming the `kanbanzai` CLI binary to `kbz`. The change touches the source directory for the main package, the build system, the MCP config templates written by `kbz init`, and all documentation references that name the binary. Because existing projects may have config files generated by a prior `kanbanzai init` run, `kbz init` must detect these stale configs and emit a targeted warning directing users to re-run init. The MCP server protocol name (`"kanbanzai"`) and the Go module path are explicitly preserved.

## Scope

### In scope

- Renaming the source directory `cmd/kanbanzai/` to `cmd/kbz/`.
- Updating the Makefile `BINARY` variable and `go build` / `go install` target paths.
- Updating the `"command"` field in the `.mcp.json` template from `"kanbanzai"` to `"kbz"`.
- Updating the `"command"` field in the `.zed/settings.json` template from `"kanbanzai"` to `"kbz"`.
- Bumping `mcpVersion` so that managed configs are rewritten on the next `kbz init --update-skills` run.
- Implementing migration detection in `kbz init`: detecting stale managed configs that still reference `"command": "kanbanzai"` and printing a human-readable warning.
- Updating `AGENTS.md`, README, and all other documentation files that reference the binary name `kanbanzai`.
- Updating the `go install` instruction in installation documentation to `go install github.com/sambeau/kanbanzai/cmd/kbz@latest`.

### Out of scope

- Changing the Go module path (`github.com/sambeau/kanbanzai`).
- Changing internal package import paths (`github.com/sambeau/kanbanzai/internal/...`).
- Changing the MCP server protocol name (`"kanbanzai"` in `mcpServers` keys and `context_servers` keys), which preserves existing tool-permission configs such as `mcp:kanbanzai:status`.
- Changing the `ServerName` constant in `internal/mcp/server.go`.
- Changing the `.kbz/` directory name.
- Changing prose references to "Kanbanzai" as the system name (as distinct from the binary name).
- Any deprecation period or continued support for a `kanbanzai` binary shim.
- Status command changes (covered by B36-F2â€“F4).

## Functional Requirements

- **FR-001:** The built binary produced by `make build` MUST be named `kbz`.
- **FR-002:** The `go build` invocation in the Makefile MUST target `./cmd/kbz`.
- **FR-003:** The `go install` invocation in the Makefile MUST target `./cmd/kbz`.
- **FR-004:** The main package entry point MUST reside at `cmd/kbz/main.go`; `cmd/kanbanzai/` MUST NOT exist.
- **FR-005:** The `.mcp.json` template written by `kbz init` MUST set `"command": "kbz"` in the `mcpServers.kanbanzai` entry.
- **FR-006:** The `.zed/settings.json` template written by `kbz init` MUST set `"command": "kbz"` in the `context_servers.kanbanzai` entry.
- **FR-007:** The `mcpVersion` constant in `internal/kbzinit/mcp_config.go` MUST be incremented so that managed config files carrying the previous version value are treated as outdated and rewritten on the next `kbz init` run.
- **FR-008:** When `kbz init` (or `kbz init --update-skills`) runs against a project whose managed `.mcp.json` contains `"command": "kanbanzai"`, it MUST print a warning message that instructs the user to re-run `kbz init` to update the config.
- **FR-009:** When `kbz init` runs against a project whose `.zed/settings.json` contains a `context_servers` entry with `"command": "kanbanzai"`, it MUST print a warning message that instructs the user to re-run `kbz init` to update the config.
- **FR-010:** The `AGENTS.md` file at the repository root MUST NOT contain any reference to `kanbanzai` as the binary name.
- **FR-011:** The project README MUST NOT contain any reference to `kanbanzai` as the binary name.
- **FR-012:** All other documentation files in `docs/` that reference the binary by name MUST use `kbz`, not `kanbanzai`.
- **FR-013:** The installation instruction exposed to users MUST read `go install github.com/sambeau/kanbanzai/cmd/kbz@latest`.
- **FR-014:** The Go module path `github.com/sambeau/kanbanzai` and all `internal/` package import paths MUST remain unchanged.
- **FR-015:** The MCP server protocol name (`"kanbanzai"` used as the key in `mcpServers` and `context_servers`) MUST remain unchanged.

## Non-Functional Requirements

- **NFR-001:** The entire test suite (`go test ./...`) MUST pass without modification to test logic â€” only test fixture strings that assert the old binary name `"kanbanzai"` as a command value are permitted to be updated.
- **NFR-002:** The binary rename MUST NOT increase the `make build` wall-clock time by more than 5% compared to the pre-rename baseline on the same machine.
- **NFR-003:** Migration warning messages MUST be printed to stdout within the normal `kbz init` output stream and MUST be human-readable without reference to internal implementation details.
- **NFR-004:** No new runtime dependency MAY be introduced by this change.

## Dependencies and Assumptions

- **DEP-001:** This feature depends on the design document `work/_project/design-kbz-cli-and-binary-rename.md` being approved before implementation begins.
- **DEP-002:** The migration warning in `kbz init` relies on the version-gating logic already present in `internal/kbzinit/mcp_config.go` (`writeJSONConfig`, `writeZedConfig`). The version bump (FR-007) is the trigger that causes the rewrite path to execute.
- **ASM-001:** The existing `mcpVersion = 1` constant will be incremented to `2`. No rollback mechanism is required; the design specifies a clean cut with no deprecation period.
- **ASM-002:** The MCP tool-permission keys (`mcp:kanbanzai:*`) in `.zed/settings.json` are controlled by the Zed IDE's schema, not by the binary name. They are therefore unaffected by this rename.
- **ASM-003:** Automated CI runs `go test ./...` on every commit. FR-001 through FR-015 can be verified entirely within the existing CI pipeline without new infrastructure.
- **ASM-004:** The `main.go` usage text already contains `kbz` (confirmed by design Â§4.1); no change to that file's content is required.
